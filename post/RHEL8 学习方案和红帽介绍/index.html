RHEL8 学习方案和红帽介绍

· 2020-07-18 · # RHEL8
前言
  • 学习是件苦差事
    • 在《活法》中有两段文字说：

      工作马马虎虎，只想在兴趣和游戏中寻觅快活，充其量只能获得一时的快感，绝不能尝到从心底涌出的惊喜和快乐，但来自工作的喜悦并不像糖果那样—放进嘴里就甜味十足，而是需要从苦劳与艰辛中渗出，因此当我们聚精会神，孜孜不倦，克服艰辛后的成就感，世上没有哪种喜悦可以类比。

      更何况人类生活中工作占据了较大的比重，如果不能从劳动中、工作中获得充实感，那么即使从别的地方找到快乐，最终我们仍然会感到空虚和缺憾。

开源精神
  • 概述：把软件程序与源代码文件一起打包提供给用户，让用户在不受限制地使用某个软件功能的基础上还可以按需进行修改，或编制成衍生产品再发布出去
    • 简单点就是：使用自由、传播自由、修改自由、衍生品自由
  • 特点：低风险、高品质、低成本、更透明
    • 低风险：所有软件，无论是专有软件还是开源软件，都存在可能导致漏洞的缺陷。论开源软件的BUG修复程度，闭源是远远不可及的。(使用开源并不是在冒险，但是开源的非托管使用却有风险)
    • 高品质：你不会把糟糕的代码发布到网上，品质自然就会高而不是让人无法理解
    • 低成本：使用开源社区推动的软件项目可以节省大量的人力、物力和财力
    • 更透明：没有哪个笨蛋会把木马、后门等放到开放的源代码中
  • GNU GPL
    • 概述：只要软件中包含了遵循GPL协议的产品或代码，该软件就必须也遵循GPL许可协议且开源、免费，因此这个协议并不适合商用软件。
    • 特点：复制自由、传播自由、收费传播、修改自由
  • BSD
    • 概述：用户可以使用、修改和重新发布遵循该许可的软件，并且可以将软件作为商业软件发布和销售。
    • 要满足下面3个条件
      • 如果再发布的软件中包含源代码，则源代码必须继续遵循BSD许可协议。
      • 如果再发布的软件中只有二进制程序，则需要在相关文档或版权文件中声明原始代码遵循了BSD协议。
      • 不允许用原始软件的名字、作者名字或机构名称进行市场推广。
  • Apache许可证版本
    • 概述：在为开发人员提供版权及专利许可的同时，允许用户拥有修改代码及再发布的自由，所以该许可协议适用于商业软件。
    • 要严格遵守下面的4个条件
      • 该软件及其衍生品必须继续使用Apache许可协议。
      • 如果修改了程序源代码，需要在文档中进行声明。
      • 若软件是基于他人的源代码编写而成的，则需要保留原始代码的协议、商标、专利声明及其他原作者声明的内容信息。
      • 如果再发布的软件中有声明文件，则需在此文件中标注Apache许可协议及其他许可协议。
  • MPL：对开发者的源代码需求和收益之间的平衡。
  • MIT：只要程序的开发者在修改后的源代码中保留原作者的许可信息即可
Linux历史
  • 1969年以前：一个没有完成的梦想：Bell,MIT与GE的‘Multics’系统
  • 1969年：Ken Thompson的小型file server system
  • 1973年：Unix的正式诞生，Ritchie等人以C语言写出第一个正式Unix核心
  • 1977年：重要的Unix分支：BSD的诞生
  • 1979年：一个措手不及的版权宣告！
    • AT&T公司宣布了对UNIX系统的商业化计划，随之开源软件业转变成了版权式软件产业
  • 1984年之一：x86架构的Minix诞生
    • 发起了GNU源代码开放计划并制定了著名的GPL许可协议
  • 1984年之二：GNU计画与FSF基金会的成立
    • 不论是什么软体，都得要进行编译成为二进位档案(binary file)后才能够执行。
  • 1985年：GNU的通用公共许可证
    • 对于GNU的最初构想 ‘建立一个自由的Unix作业系统’来说，有这些优秀的程式是仍无法满足， 因为，当下并没有‘自由的Unix核心’存在…所以这些软体仍只能在那些有专利的 Unix平台上工作～～一直到Linux的出现
  • 1987年：gcc编译器发布
    • 这使得程序员可以基于该编译器编写出属于自己的开源软件
  • 1988年：图形介面XFree86计画
  • 1991年：芬兰大学生Linus Torvalds的一则简讯
    • Linux的操作系统诞生
  • 1994年：Liunx系统内核+源码+程序
    • 发布了红帽系统并开始出售技术服务
  • 1998年：HP、IBM、Intel推动开放源代码软件发展
  • 2017年：Linux内核已经发展到了4.13版本
  • 2020年：云原生、区块链高速发展
常见的Linux系统版本

  • 红帽企业系统（RedHatEnterpriseLinux,RHEL）系统

  • 社区企业操作系统（Community Enterprise Operating System，CentOS）系统

  • 由红帽公司发布的桌面版系统套件Fedora（目前已经不限于桌面版）系统

  • 在全球范围内有着不错的声誉及市场占有率的openSUSE系统

  • 具有极高的自定制性，操作复杂的Gentoo系统

  • 稳定性、安全性强，提供了免费的基础支持的Debian系统

  • 是一款派生自Debian的操作系统之Ubuntu系统

  • 关于系统软件是否更新：不论是学习Linux还是编程语言，都应该选择当前稳定且最新的版本作为学习环境。

  • 稳定或者最新

    • 稳定：无论是进行开发还是运维，稳定压倒一切。
    • 最新：老版本可能会有更大的概率存在安全漏洞或者功能缺陷，而新版本不仅出现漏洞的概率小，而且即便出现漏洞，也会快速得到众多开源社区和企业的响应并更快地修复。
红帽认证

  • RHCSA：红帽认证系统管理员（Red Hat Certified System Administrator）属于Linux系统的初级认证

  • RHCE：红帽认证工程师（Red Hat Certified Engineer）属于Linux系统的中级水平认证

  • RHCA：红帽认证架构师（Red Hat Certified Architect）属于Linux系统的最高级别认证
